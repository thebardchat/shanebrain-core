# Module 3.5 Hints — Daily Briefing

## Level 1 — General Direction

- This module uses three MCP tools: `daily_note_add`, `daily_note_search`, and `daily_briefing`
- The exercise adds three types of notes (journal, todo, reminder) then generates a briefing
- You need the MCP server running on port 8100 — check with the health check first
- The daily briefing is generated by Ollama, so it needs a few seconds to process
- If the briefing comes back empty, Ollama might be loading the model — try again

## Level 2 — Specific Guidance

- **"Cannot reach MCP server"**: Make sure the MCP server is running. Check with `curl -s http://localhost:8100/mcp` or run the shared health check at `shared\utils\mcp-health-check.bat`
- **"daily_note_add failed"**: Check that you're passing valid JSON. The `content` field is required. The `note_type` field should be one of: journal, todo, reminder, reflection
- **"daily_note_search returns nothing"**: You need to add notes first (Tasks 1-3). Search uses semantic matching, so your query doesn't need exact words — just related meaning
- **"daily_briefing returns error"**: Ollama needs to be running and have a model loaded. The briefing tool calls Ollama to generate the summary. Check `system_health` for Ollama status
- **Verify fails on "briefing contains actual text"**: The briefing response must have real content, not just an error message. If Ollama is slow, wait 30 seconds and try again

## Level 3 — The Answer

Complete sequence to pass verification:

**Step 1: Make sure MCP is reachable**
```
cd shared\utils
mcp-health-check.bat
```

**Step 2: Run the exercise**
```
cd phases\phase-3-everyday\module-3.5-daily-briefing
exercise.bat
```

The exercise will:
1. Add a journal entry with mood tag (grateful or focused)
2. Add a todo note
3. Add a reminder note
4. Generate a daily briefing from your notes

**Step 3: If tasks fail individually**
```
:: Manual journal entry
python shared\utils\mcp-call.py daily_note_add "{\"content\":\"Good day at work today\",\"note_type\":\"journal\",\"mood\":\"grateful\"}"

:: Manual todo
python shared\utils\mcp-call.py daily_note_add "{\"content\":\"Send estimate to client\",\"note_type\":\"todo\"}"

:: Manual reminder
python shared\utils\mcp-call.py daily_note_add "{\"content\":\"Truck inspection next week\",\"note_type\":\"reminder\"}"

:: Manual briefing
python shared\utils\mcp-call.py daily_briefing
```

**Step 4: Verify**
```
verify.bat
```

**If verify still fails on briefing text**: Ollama may need time to load the model. Run `python shared\utils\mcp-call.py daily_briefing` manually and wait up to 60 seconds for a response.
